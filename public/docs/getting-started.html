Rozpoczęcie pracy

Instalacja

Aby rozpocząć pracę z frameworkiem BubbleCSS należy umieścić natępujące linki w znaczniku <head>.

  • data-base-path - ścieżka katalogu głównego projektu. Dla stron w katalogu głównym projektu przeważnie ./, dla podstron w podfolderach ../.
  • data-logo-path - nazwa pliku logo w katalogu /assets/images.
    • <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Page title</title>
    <link rel="stylesheet" href="https://adambru.github.io/BubbleCSS/main.css">
    <script src="https://adambru.github.io/BubbleCSS/script.js" type="module" data-base-path="./" data-logo-path="bubblecss-logo-aqua.png"><script>
</head>

    Alternatywnym sposobem korzystania z BubbleCSS jest pobranie paczki BubbleCSS.zip, rozpakowanie jej i umieszczenie folderów w sposób opisany w sekcji Struktura projektu.

    Struktura projektu

    Aby framework działał poprawnie, musi zostać zachowana właściwa organizacja plików.

    Struktura folderów i plików

    / (project root)
├─ assets
│  └─ images
│  └─ config.css
│  └─ style.css
├─ components
├─ index.html
├─ subfolder1
│  └─ subpage1.html
└─ subfolder2
   └─ subpage2.html

    Uwaga! TODO: Uwaga! badge W przypadku korzystania z pobranej paczki należy wyłącznie przenieść foldery w odpowiednie miejsca według schematu nie zmieniając ich podfolderów.

    Konfiguracja

    Aby dostosować wygląd stron, należy edytować pliki config.css i style.css w folderze /assets. Do każdego dokumentu należy w znaczniku <head> dodać link do plików.

    *.html

    ...
    <link rel="stylesheet" href="/assets/config.css">
    <link rel="stylesheet" href="/assets/style.css">
</head>

    config.css

    :root {
    /* Definicje kolorów */
    --color-main: #ff0000;
    --color-accent-1: #00ff00;
    --color-accent-2: #0000ff;

    /* Wartość zaokrąglenia rogów; wartość domyślna: 30px */
    --round-value: ;
}
